ClientCommunicationSignal

This item only works when running on the client. Client

A client communication signal contains all the members, remote signals and remote properties binded to it's server counter part. 

-- On the server:
local communicationSignal = CommunicationSignal.new("ServersideService")
communicationSignal:Bind("Test", 123)
communicationSignal:Bind("T", {1, 2, 3})
communicationSignal:Bind("Signal", RemoteSignal.new())
communicationSignal:DispatchToClient()

-- On the client:
local communicationSignal = CommunicationSignal.GetDispatchedSignal("ServersideService")

print(
	communicationSignal.Members.Test,  --> 123
	communicationSignal.Members.T, --> {1, 2, 3}
	communicationRemote.RemoteSignals.Signal 
)
note

Remote signals and remote properties exposed to the client are represented differently on the client. They are represented as client remote properties and client remote signals, except they are also binded to their serverside counter parts.

For e.g:

-- On the server
local communicationSignal = CommunicationSignal.new("ServersideService")
communicationSignal:Bind("Bo", RemoteSignal.new())
communicationSignal:DispatchToClient()

-- On the client
local communicationSignal = CommunicationSignal.GetDispatchedSignal("ServersideService"):expect() 
print(ClientRemoteSignal.IsA(communicationSignal.RemoteSignals.Bo)) --> true

GetDispatchedSignal#

This is a yielding function. When called, it will pause the Lua thread that called the function until a result is ready to be returned, without interrupting other scripts. Yields
</>
ClientCommunicationSignal.GetDispatchedSignal(
identifier:Â string,
timeout:Â numberÂ |Â nil
) â†’Â Promise

Returns a promise which is resolved with the serverside dispatched signal if found. If timeout is specified, then the method will wait for timeout seconds for the serverside dispatched signal to exist before assuming that it doesn't and rejecting with a nil value. 

-- On the server:
local signal = CommunicationSignal.new("Signal")
signal:Bind("bo", true)

-- On the client:
CommunicationSignal.GetDispatchedSignal("Signal"):andThen(function(signal)
	print(signal.Members.bo) --> true
end)
tip

Calling this method for the same dispatched signal will return cached results.

For e.g:

-- On the server:
local signal = CommunicationSignal.new("Signal")
signal:Bind("bo", true)
signal:DispatchToClient()

-- On the client:
local s = CommunicationSignal.GetDispatchedSignal("Signal"):expect()
local o = CommunicationSignal.GetDispatchedSignal("Signal"):expect()
print(s == o)
warning

The returned communication signal will also be destroyed once the dispatched signal's serverside counterpart is destroyed.

For e.g:

-- On the server:
local signal = CommunicationSignal.new("Signal")
signal:DispatchToClient()
task.delay(5, function() signal:Destroy() end))

-- On the client
local sig = CommunicationSignal.GetDispatchedSignal("Signal"):expect()
print(typeof(sig.Members)) --> "table" 
task.wait(5) -- wait for it to be destroyed on the server
print(typeof(sig.Members)) --> "nil"

Build#

</>
ClientCommunicationSignal:Build() â†’Â table

A method which builds the communication signal for client-side use.

For e.g:

-- On the server:
local signal = CommunicationSignal.new("Signal")
signal:Bind("Table", {1,2,3})
signal:DispatchToClient()

-- On the client:
local clientCommunicationSignal = CommunicationSignal.GetDispatchedSignal("Signal"):expect()
local builtDispatchedSignal = clientCommunicationSignal:BuildDispatchedSignal()

-- Normally to access members you would do:
print(clientCommunicationSignal.Members.Table) --> {1, 2, 3}

-- Via dispatched signal:
print(builtDispatchedSignal.Table) -->  {1, 2, 3}
tip

Calling this method again will return cached results.

For e.g:

local s = clientCommunicationSignal:Build()
local o = clientCommunicationSignal:Build()
print(s == o) --> true
